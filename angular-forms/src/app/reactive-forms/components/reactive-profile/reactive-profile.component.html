📚 Advanced Reactive Forms - Key Concepts

1. Nested FormGroup (Hierarchical Structure)

Group related fields within a parent FormGroup: 

address: this.formBuilder.group({{ '{' }}
  street: ['', Validators.required],
  city: ['', Validators.required],
  zipCode: ['', Validators.required]
{{ '}' }})

Access nested values: form.get('address.street')?.value

2. FormArray (Dynamic Collections)

Use FormArray for dynamic lists of controls: 

skills: this.formBuilder.array([
  this.formBuilder.control('', Validators.required)
])

Methods:

  • push() - Add control to array
  • removeAt(i) - Remove control at index
  • clear() - Remove all controls
  • insert(i, control) - Insert at specific index
3. Custom Validators

The phone field uses a custom validator that validates phone number format: 

phone: ['', [Validators.required, phoneValidator()]]

Custom validators can implement complex logic beyond built-in validators.

4. setValue vs PatchValue vs Reset
  • setValue() - Set all controls (throws error if missing)
  • patchValue() - Set some controls (optional)
  • reset() - Reset to initial values
5. Real-World Use Cases

This pattern is perfect for:

  • Multi-step forms with sections
  • Dynamic forms that add/remove fields based on user input
  • Complex business data structures
  • Forms with repeating sections (line items, team members, etc.)
