📚 Reactive Forms - Key Concepts

1. FormControl & FormGroup

FormControl: Manages a single form input 

const emailControl = new FormControl('', Validators.required);

FormGroup: Groups multiple controls together 

const loginForm = new FormGroup({
  email: new FormControl('', Validators.required),
  password: new FormControl('', Validators.minLength(6))
});
2. FormBuilder - Simplified Form Creation

FormBuilder provides a shorter syntax for creating forms: 

this.form = this.formBuilder.group({
  email: ['', [Validators.required, Validators.email]],
  password: ['', Validators.minLength(6)]
});
3. Validators

Built-in Validators:

  • Validators.required - Field cannot be empty
  • Validators.email - Must be valid email format
  • Validators.minLength(n) - Minimum character count
  • Validators.maxLength(n) - Maximum character count
  • Validators.pattern(regex) - Must match pattern
4. Async Validators

Async validators run asynchronous checks (API calls): 

email: ['', {
  validators: Validators.email,
  asyncValidators: emailAvailabilityValidator(),
  updateOn: 'blur'
}]

In this form, the email field has an async validator that checks if the email is already registered. The form status becomes "PENDING" while the check is running.

5. Form Status and Value Changes

Subscribe to form changes reactively:

  • form.valueChanges - Emits when any value changes
  • form.statusChanges - Emits when validation status changes
  • form.valid - Boolean indicating if form is valid
  • form.pending - Boolean indicating if async validators are running
6. When to Use Reactive Forms

✅ Best for: Complex forms, dynamic fields, unit testing, custom validation
❌ Avoid for: Very simple forms where template-driven is overkill
